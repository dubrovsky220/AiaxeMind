# Sprint planning

Product vision, modules, tech stack, and phased roadmap live in `**[../description.md](../description.md)**`. This folder holds **one file per sprint**, aligned with the **revised 8-sprint plan** optimized for MVP delivery.

## Sprint Structure: MVP-First Approach

**Sprints 1-3 = Complete MVP** (4-5 weeks)

- Sprint 1: Core RAG foundation
- Sprint 2: Socratic teaching engine (main value)
- Sprint 3: Quality, testing, documentation (production-ready)

**Sprints 4-8 = Enhancements** (5-7 weeks)

- Sprint 4: Improved RAG (re-ranking, optimization)
- Sprint 5: Multi-modal (images)
- Sprint 6: Paper Tutor mode
- Sprint 7: Personalized quizzes
- Sprint 8: Workspace management & production features

## How this connects to GitHub Issues

1. Before each sprint: read the sprint file; adjust scope if needed
2. Break the sprint into **issues** with labels `sprint:N` (N = 1…8), `type:`*, `priority:*` — see `[../agile.md](../agile.md)`
3. Implementation details (schemas, API shapes, prompts) belong in **issues and PRs**, not in separate design docs
